# <a name="azure-purview-product-glossary"></a>Azure Purview 製品用語集

Azure Purview 全体で使用される用語集を以下に示します。

## <a name="annotation"></a>注釈
用語集の用語や分類など、Azure Purview のデータ資産に関連付けられている情報。 適用された注釈は、検索で使用して、データ資産の検出に役立てることができます。 
## <a name="approved"></a>Approved
その要求の状態を変更する権限を持つ指定の個人やグループによって十分であると受け取られている、あらゆる要求に与えられる状態。 
## <a name="asset"></a>資産
Azure Purview データ カタログ内に格納されるあらゆる単一のオブジェクト。
> [!NOTE]
> カタログ内の単一のオブジェクトは、潜在的にストレージ内の数多くのオブジェクトを表すことがあります。たとえば、リソース セットはアセットですが、ストレージ内の数多くのパーティション ファイルで構成されています。
## <a name="azure-information-protection"></a>Azure Information Protection
情報を分類して保護するためのドキュメントと電子メールのラベル付けをサポートするクラウド ソリューション。 ラベル付けされた項目は、暗号化によって保護したり、透かしでマークしたり、特定のアクションまたはユーザーに制限したりできます。ラベルはその項目にバインドされます。 このクラウドベースのソリューションでは、制限を適用するのに Azure Rights Management Service (RMS) に依存します。 
## <a name="business-glossary"></a>ビジネス用語集
組織が主なビジネス用語とその定義を記述するために使用する、専門用語の検索可能リスト。 ビジネス用語集を使用すると、組織全体のデータ使用の一貫性を保つことができます。 
## <a name="classification-report"></a>分類レポート
スキャンされたデータに関する主な分類の詳細を示すレポート。  
## <a name="classification"></a>分類
資産の属性または列を識別するために使用される注釈の種類 ("Age (年齢)"、"Email Address (メール アドレス)"、"Street Address (番地)" など)。 これらの属性は、スキャン中に割り当てるか、手動で追加できます。 
## <a name="classification-rule"></a>分類規則
分類規則とは、コンテンツが指定されたパターンに一致するときに、スキャンされたデータを分類する方法を決定する、一連の条件のことです。
## <a name="classified-asset"></a>分類済み資産
Azure Purview によって自動スキャン中にスキーマが抽出され、分類が適用される資産。 スキャン ルール セットによって、どの資産が分類されるかが決まります。 その資産が分類の候補と見なされており、かつスキャン時に分類が適用されていない場合でも、資産は分類済み資産と見なされます。
## <a name="column-pattern"></a>列パターン
分類規則に含まれる、一致させる必要がある列名を表す正規表現。
## <a name="contact"></a>Contact
データ カタログ内のエンティティに関連付けられている個人 
## <a name="control-plane-operation"></a>コントロール プレーン操作
ロールベースのアクセス制御や Azure ポリシーなど、Azure Resource Manager エンドポイントに送信される、サブスクリプション内のリソースを管理する操作。 
## <a name="credential"></a>資格情報
アクセス制御システムで使用される本人確認方法またはツール。 資格情報は、データ資産へのアクセスを許可する目的で、個人またはグループを認証するのに使用できます。 
## <a name="data-catalog"></a>データ カタログ
顧客がデータ資産内の資産のメタデータを表示して管理できる、Azure Purview の機能。
## <a name="data-map"></a>データ マップ
スキャン、系列の表示、移動など、顧客がデータ資産を管理するための Azure Purview の機能。
## <a name="data-pattern"></a>データ パターン
データ フィールドに格納されているデータを表す正規表現。 たとえば、従業員 ID のデータ パターンは Employee{GUID} となる場合があります。
## <a name="data-plane-operation"></a>データ プレーン操作
資産の編集や用語集の用語の作成など、特定の Azure Purview インスタンス内での操作。 各インスタンスには、ユーザーが実行できるデータ プレーン操作を制御する ""データ閲覧者""、""データ キュレーター"" などの定義済みのロールがあります。
## <a name="discovered-asset"></a>検出された資産
スキャン プロセス中に Azure Purview によってデータ ソースで識別される資産。 検出された資産の数には、リソース セットのグループ化前のすべてのファイルやテーブルが含まれます。
## <a name="distinct-match-threshold"></a>個別の一致しきい値
スキャナーがデータ パターンを実行する前に、列に含まれている必要がある個別のデータ値の合計数です。 たとえば、従業員 ID に対して個別の一致しきい値を 8 に設定するには、サンプリングされた値のうち、従業員 ID に設定されたデータ パターンと一致する一意のデータ値が、列内に少なくとも 8 つ存在している必要があります。
## <a name="expert"></a>Expert
データ資産や用語集の用語の完全なコンテキストを理解している組織内の個人。
## <a name="full-scan"></a>フル スキャン
データ ソースの選択したスコープ内のすべての資産が処理されるスキャン。
## <a name="fully-qualified-name-fqn"></a>完全修飾名 (FQN)
データ ソース内の資産の場所を定義するパス。  
## <a name="glossary-term"></a>用語集の用語
ある組織に固有の概念を定義するビジネス用語集のエントリ。 用語集の用語には、同意語、頭字語、関連用語に関する情報が含まれる場合があります。
## <a name="incremental-scan"></a>増分スキャン
前回の正常なスキャン以降に作成、変更、または削除された資産を検出して処理するスキャン。 増分スキャンを実行するには、ソースで少なくとも 1 回のフル スキャンが実行されている必要があります。
## <a name="ingested-asset"></a>取り込まれた資産
スキャンと分類 (該当する場合) が実行され、Azure Purview データ マップに追加された資産。 取り込まれた資産は、自動スキャンや外部接続 (Azure Data Factory や Azure Synapse など) を通じて、データ カタログ内で検出されて利用できるようになります。
## <a name="insights"></a>洞察
データに関する情報が要約されたレポートを表示できる、Azure Purview 内の領域。
## <a name="integration-runtime"></a>統合ランタイム
データ ソースでのスキャンに使用されるコンピューティング インフラストラクチャ。
## <a name="lineage"></a>系列
データが移動元から移動先に移動する際の変換方法とフロー。 データ資産全体にわたってこのフローを把握しておくと、自分の組織のデータ履歴を確認し、トラブルシューティングや影響分析に役立てることができます。 
## <a name="management-center"></a>管理センター
接続、ユーザー、ロール、資格情報を管理できる Azure Purview 内の領域。
## <a name="minimum-match-threshold"></a>最小の一致しきい値
分類を適用するためにスキャナーによって検出される必要がある、列内の個別のデータ値の最小一致率。 

たとえば、従業員 ID の最小一致しきい値が 60% の場合は、ある列でサンプリングされたデータのうち、個別のすべての値の 60% が、従業員 ID に設定されたデータ パターンと一致する必要があります。 ある列でスキャナーによって 128 個の値がサンプリングされ、その列で個別の値が 60 個見つかった場合、その分類が適用されるには、36 個以上の個別の値 (60%) が従業員 ID のデータ パターンと一致する必要があります。
## <a name="on-premises-data"></a>オンプレミスのデータ
クラウド内やサービスとしてのソフトウェア (SaaS) などではなく、顧客が管理するデータ センター内にあるデータ。 
## <a name="owner"></a>所有者
データ資産の管理を担う個人またはグループ。
## <a name="pattern-rule"></a>パターン ルール
Azure Purview で資産をリソース セットとしてグループ化し、カタログ内に表示する方法をオーバーライドする構成。
## <a name="purview-instance"></a>Purview インスタンス
単一の Azure Purview リソース。 
## <a name="registered-source"></a>登録済みソース
Azure Purview インスタンスに追加され、データ カタログの一部として管理されるようになったソース。 
## <a name="related-terms"></a>関連用語
組織内の他の用語にリンクされている用語集の用語。  
## <a name="resource-set"></a>リソース セット
ストレージ内の多数のパーティション分割されたファイルまたはオブジェクトを表す単一の資産。 たとえば、Azure Purview では、Apache Spark の分割された出力を、個々のファイルごとに一意のアセットではなく、単一のリソース セットとして格納します。 
## <a name="role"></a>職務
Azure Purview インスタンス内のユーザーに割り当てられたアクセス許可。 Purview データ キュレーターや Purview データ閲覧者などのロールによって、製品内で実行できる処理が決まります。
## <a name="scan"></a>スキャン
ソースまたはソースのセットを分析し、そのメタデータをデータ カタログに取り込む Azure Purview のプロセス。 スキャンは、手動で実行するか、スキャン トリガーを使用してスケジュールに基づいて実行できます。 
## <a name="scan-ruleset"></a>スキャン ルールセット
スキャンによってカタログに取り込まれるデータ型と分類を定義する一連のルール。 
## <a name="scan-trigger"></a>スキャン トリガー
スキャンの実行の繰り返しを決定するスケジュール。
## <a name="search"></a>検索
キーワードに一致する資産の一覧を返す Azure Purview のデータ検出機能。 
## <a name="search-relevance"></a>検索の関連性
検索結果が返される順番を決定するデータ資産のスコアリング。 資産の関連性スコアは、複数の要素によって決まります。
## <a name="self-hosted-integration-runtime"></a>セルフホステッド統合ランタイム
オンプレミスのマシンまたはプライベート ネットワーク内の仮想マシンにインストールされた統合ランタイム。オンプレミスまたはプライベート ネットワーク内のデータへの接続に使用されます。
## <a name="sensitivity-label"></a>秘密度ラベル
組織のデータを分類して保護する注釈。 Azure Purview では、Microsoft Information Protection と連携して、秘密度ラベルを作成します。 
## <a name="sensitivity-label-report"></a>秘密度ラベル レポート
データ資産全体に適用される秘密度ラベルの概要。 
## <a name="service"></a>サービス
スタンドアロン機能を提供し、サブスクリプションまたはライセンスによって顧客が利用できる製品。 
## <a name="source"></a>ソース
データが格納されるシステム。 ソースは、クラウドやオンプレミスなどのさまざまな場所にホストできます。 ソースを登録してスキャンすることで、Azure Purview で管理できるようにします。 
## <a name="source-type"></a>変換元の型
Azure Purview インスタンスで使用される登録済みソースの分類 (Azure SQL Database、Azure Blob Storage、Amazon S3、SAP ECC など)。 
## <a name="steward"></a>スチュワード
用語集の用語の標準を定義する個人。 割り当てられたエンティティの品質基準、命名法、規則を維持する役割を担います。 
## <a name="term-template"></a>用語テンプレート
用語集の用語に含まれる属性の定義。 ユーザーは、システム定義の用語テンプレートを使用するか、独自の用語テンプレートを作成してカスタム属性を含めることができます。
